  1. Introducción

El presente documento especifica las políticas de backup para el sistema MakeEvents. Describe los datos del servidor que se resguardarán, el personal autorizado a operar los backups y restauraciones, los datos a resguardar, el momento en que se ejecutan lo backups, el manejo de los datos almacenados, y las responsabilidades de los operadores y demás personas involucradas.

 

  1. Alcance

Aplica a todos los datos alojados en servidores y base de datos bajo resguardo acorde a los objetivos de seguridad de los datos establecidos por el área.

 

  1. Niveles de Backup que se realizarán

Se realizará dos tipos de backup, Completo y Diferencial

 

  1. Archivos (Críticos) que deben tener copia de Respaldo

-       Software aplicativo, considerando tanto los programas fuentes, y cualquier otro software o procedimiento que también trabaje con los datos, para producir los resultados con los cuales trabaja el usuario final.

-       Datos y estructura de datos (Bases de Datos, Índices, tablas de validación, contraseñas, usuarios, roles, configuraciones y todo archivo necesario para el funcionamiento de los Sistemas de Información del Aplicativo y la pronta recuperación de los mismos en caso de fallas).

 

  1. Especificaciones y normas para la elaboración de Backup en medios extraíbles

Los backups deberán estar almacenados en DVD de primera marca,con etiquetas que contengan la siguiente información mínima:

- Nombre del Sistema.

- Versión / Año.

- Ubicación de las fuentes(Programas Fuente).

- Ubicación de los registros de la Base de Datos(Archivos Críticos).

La ubicación de resguardo de dichos backups será la Av. Kennedy 1402 ubicado al frente del parque Kennedy.

  1. Procedimiento General de Resguardo

Siempre que sea posible, el resguardo de los datos se hará de la siguiente forma, y con el respectivo responsable.

-       Los viernes a las 17:30 de cada mes se hará un backup completo de la base de datos.

-       Los miércoles a las 16:00 de cada quincena de mes se hará un backup diferencial

 

  1. Procedimiento de Recuperación

Realizado por el encargado de la recuperación, entonces bajo las consideraciones del procedimiento de Resguardo se podrá recuperar una versión para el último mes. Ante la posible pérdida de archivos se podrá acudir a los backups diferenciales, y ante una pérdida total del servidor se podrá acudir a los respaldos completos. De esta forma es posible recomponer el sistema por completo.
